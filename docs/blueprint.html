1. Project Overview

AFCS Smart Campus is a comprehensive school management operating system built for Nigerian secondary schools. It digitizes and automates:

  • Staff & Student Attendance — QR-based check-in, late detection, batch operations, department/class breakdowns
  • Duty Roster Management — 8 duty types, auto-rotation, status tracking, Telegram notifications
  • Muster Parade System — Sessions, briefings, tasks with acknowledgement workflow
  • AI Timetable Generator — Constraint-satisfaction algorithm that distributes subjects evenly Mon-Fri with Friday 30-min periods
  • Daily Reporting — Inspection reports, student activity reports, AI-generated summaries
  • Telegram Bot — 25+ commands, interactive task management, real-time push notifications
  • Multi-Channel Notifications — Telegram (primary), WhatsApp Cloud API, SMS (Termii/Africa's Talking)
  • Automation Engine — 13 scheduled rules for duty notification, check-in reminders, daily digests
  • AI Assistant — OpenAI/Gemini-powered chat with database function calling
  • Role-Based Access — Commandant, Admin, Teacher, Support with distinct dashboards

5. Authentication System

Production Flow

  1. User visits /login, enters Staff ID or email
  2. POST /api/auth/lookup-staff finds staff record
  3. User enters password; supabase.auth.signInWithPassword() called
  4. Supabase sets HTTP-only cookies (session)
  5. AuthContext listens to onAuthStateChange
  6. Redirect by role: commandant→/dashboard, teacher→/teacher-dashboard
  7. AuthGate protects all routes

Dev Mode Bypass

Set NEXT_PUBLIC_DEV_MODE=true. A staff selector UI saves to localStorage. The DevAuthInterceptor component injects x-auth-email header to all fetch calls, and API routes check this fallback for authorization.

8. Automation Engine

Architecture

Entry point: POST /api/automation/engine. Triggered by Vercel Cron (6:00 AM UTC), Telegram /automate command, or manual button. Each rule checks its time window before executing. Duplicate prevention via last_run_at with 30-min cooldown.

11. Deployment Guide

Platform: Vercel

  1. Push code to GitHub repository
  2. Create new Vercel project from GitHub repo
  3. Set all environment variables in Vercel Project Settings
  4. Build command: next build (default)
  5. Deploy triggers automatically on push to master

Supabase Setup

  1. Create Supabase project
  2. Enable uuid-ossp extension in SQL Editor: CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
  3. Run migration files sequentially (001 through latest)
  4. Configure Authentication providers (email/password)
  5. Copy project URL, anon key, service role key to Vercel env vars

Telegram Bot Setup

  1. Create bot via @BotFather on Telegram
  2. Copy token and set TELEGRAM_BOT_TOKEN env var
  3. After deployment, navigate to Settings → Telegram in the app
  4. Click "Set Webhook" to register the webhook URL
  5. Or manually: https://api.telegram.org/bot{TOKEN}/setWebhook?url={APP_URL}/api/telegram/webhook

WhatsApp Cloud API Setup

  1. Create Meta Business Account + WhatsApp Business Account
  2. Set up webhook at {APP_URL}/api/whatsapp/webhook with verify token
  3. Create message templates (duty, task, period, broadcast)
  4. Set env vars with template names

Cron Jobs

vercel.json defines a single cron: 0 6 * * * (daily 06:00 UTC). The automation engine internally checks each rule's time window.

  1. Initialize Next.js project with TypeScript and App Router
  2. Set up Tailwind CSS v4
  3. Create Supabase project, enable uuid-ossp
  4. Create core migrations: departments, staff, settings
  5. Build staff CRUD page and API
  6. Set up authentication with Supabase Auth
  7. Create AuthContext, AuthGate, role-based routing
  8. Build responsive sidebar navigation

Phase 2: Attendance (Week 3-4)

  1. Create staff_attendance table migration
  2. Build check-in/out API routes
  3. Build CheckInForm with QR code scanning
  4. Build attendance dashboard with stats/charts
  5. Create student schema (classes, students, student_attendance)
  6. Build batch student check-in by class
  7. Build student attendance dashboard with class breakdowns
  8. Add late detection and attendance cutoff logic

Phase 3: Duties & Reports (Week 5-6)

  1. Create duty schema (duty_types, duty_rosters, daily_reports)
  2. Build duty type CRUD
  3. Build duty roster management with auto-rotation
  4. Build daily report submission form
  5. Build reports dashboard with filters and export
  6. Add student activity reports for class teachers

Phase 4: Parade & Tasks (Week 7)

  1. Create parade schema (sessions, briefings, tasks)
  2. Build parade session CRUD with status tracking
  3. Build task assignment with priority and deadlines
  4. Add acknowledgement workflow
  5. Build task list page with filters

Phase 5: Timetable Generator (Week 8-10)

  1. Create timetable schema (terms, subjects, time_slots, rooms, entries, class_subjects)
  2. Build subject CRUD with difficulty tiers
  3. Build teacher-subject assignment
  4. Build class-subject assignment
  5. Build time slots configuration
  6. Implement AI timetable generation algorithm:
    • Load classes, subjects, teachers, slots, rooms
    • Build class requirements with period counts
    • Greedy assignment with shuffling
    • Morning priority for hard subjects
    • Double-period support for sciences
    • Teacher load balancing
    • Conflict detection and reporting
    • Quality scoring (distribution, morning ratio, load balance, conflict-free rate)
  7. Build timetable viewer with color-coded cells
  8. Add Friday 30-min period support
  9. Add generation history with publish workflow

Phase 6: Telegram Bot (Week 11-12)

  1. Create Telegram bot with @BotFather

  6. Implement report submission (/report)

  8. Add inline keyboards for interactive actions

  3. Implement each rule handler:
