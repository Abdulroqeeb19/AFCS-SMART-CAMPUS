The Problem

Nigerian schools face critical operational challenges:

  • Paper-based record keeping — Attendance, duties, and reports are handwritten, lost easily, and nearly impossible to audit. A typical Nigerian school processes 500+ attendance records daily on paper.
  • No real-time visibility — School administrators (commandants, principals) lack live data on who is present, who is on duty, or what tasks are pending — leading to accountability gaps.
  • Manual timetable generation — Creating conflict-free timetables for 30+ teachers across multiple classes takes 3-5 days of manual work and frequently contains room/teacher/subject clashes.
  • Fragmented communication — WhatsApp groups, physical notice boards, word-of-mouth. No unified system for task assignment, acknowledgment, or follow-up.
  • No accountability framework — Task assignment and follow-up are informal. There is no digital audit trail for duty assignments, parade briefings, or attendance overrides.
  • Limited oversight — Proprietors, education boards, and ministry officials cannot access school performance data remotely. No dashboards for attendance trends or staff performance.
  • No student leadership management — Prefect assignments are manual, inconsistently tracked, and not integrated with the attendance or behavior monitoring system.

Market Gap

Existing solutions (Edusuite, SchoolTry, SkoolMedia, Eduplana) focus on academic records — grades, transcripts, fees, and learning management. None provide integrated school operations management: the daily workflow of running a boarding school — attendance tracking, duty rotations, parade management, real-time staff communication, automated reporting, and prefect management. AFCS Smart Campus uniquely fills this gap as the only platform combining operations management + academic scheduling + multi-channel communications + automation.

The Solution

AFCS Smart Campus is a mobile-first, cloud-native school operating system with 12 integrated modules:

  1. Smart Attendance (Staff & Student) — QR code and manual check-in/out with late detection, department and class breakdowns, real-time dashboard with auto-refresh. Configurable late cutoff and closing times.
  2. QR Code Scanner — Native barcode detection with pure-JavaScript fallback. Camera-based scanning for staff ID cards and student IDs. Now with improved reliability, auto-start, and stale-closure fixes.
  3. Duty Roster Engine — 8 duty types (Morning Talk, Assembly Duty, Inspection/Report, Time, Sanitary, Labour, Dining Hall, Sports). Auto-rotation algorithm, status tracking, Telegram notifications for daily assignments.
  4. AI Timetable Generator — Constraint-satisfaction algorithm producing optimized, conflict-free timetables. Supports double periods, Friday 30-min periods, morning priority for hard subjects, teacher-subject-class-room constraints. Quality metrics dashboard.
  5. Muster Parade Management — Digital parade session tracking, briefing delivery, task assignment with acknowledgment workflow, priority/deadline tracking, and live response feed for commandant.
  6. Prefect Roles System — 18 prefect roles (Head Boy/Girl through Time Prefect). Dropdown assignment on student cards, prefect badges, and filter-by-prefect search. Role-based leadership tracking.
  7. Daily Intelligence Reports — Structured reporting with activity descriptions, challenges, and suggestions. AI-generated summaries and trend analysis. Student activity reports by class teachers.
  8. Multi-Channel Communications — Telegram bot (25+ commands including /automate, /duty, /tasks, /attendance), WhatsApp Cloud API with templates, SMS via Termii + Africa's Talking. Cascade fallback: Telegram → SMS → WhatsApp → Queue.
  9. Automation Engine — 13 scheduled rules handling duty roster notifications, check-in reminders, absentee alerts (configurable threshold), next-period reminders, daily summary broadcasts, parade auto-close, duty auto-assign, assembly reminders, scheduled broadcast processing, and end-of-day digest. All notifications now delivered via Telegram.
  10. Global Search — Ctrl+K or sidebar search across all 30+ platform features. Keyboard-navigable modal with section grouping.
  11. AI Assistant — Natural language query interface with OpenAI (GPT-4o), Google Gemini, and Ollama (Qwen3:4b free local option). Database function calling for instant attendance, timetable, and staff insights.
  12. Notifications Hub — Full notification history with sent/failed/queued status. Print queue for offline distribution. SMS queue with retry mechanism.

Security Architecture

  • Row-Level Security (RLS) — Every database query enforces per-user access policies. Staff see only their own data.
  • Role-Based Access Control — 4-tier hierarchy: Commandant (100) > Admin (80) > Teacher (40) > Support (20). Granular action-level permissions.
  • Service Role Isolation — Admin API operations use a separate service-role client with elevated privileges, never exposed to client-side code.
  • Complete Audit Trail — All overrides, staff edits, and task changes logged with user identity and timestamp.
  • Zero-Knowledge Authentication — Supabase Auth handles all credentials. We never see or store passwords.
  • Request Validation — Body size limits, type checking, and input sanitization on all write endpoints.
